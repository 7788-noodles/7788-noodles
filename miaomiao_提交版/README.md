# 喵喵记账小程序

一个基于微信小程序 + Flask 后端的轻量级 AI 记账项目，支持：

- 文字记账
- 语音记账
- 百度语音识别转文字
- AI 智能分类
- 账单增删改查
- 消费统计
- AI 消费分析与建议

---

## 项目结构

```text
miniprogram/      微信小程序前端
server/           Flask 后端
pages/index/      首页（文字记账、语音记账、最近账单）
pages/bills/      账单列表页
pages/edit/       编辑账单页
pages/stats/      统计页

```
## 运行环境

- 微信真机测试环境

## 安装与运行
1. 克隆项目
git clone 你的仓库地址
cd 你的项目目录
2. 配置后端密钥
进入 server/ 目录，复制配置模板文件：

macOS / Linux：
cp config.example.py config.py

Windows：
手动复制一份：
config.example.py
重命名为：
config.py

然后在 server/config.py 中填写你自己的密钥：

API_KEY = "你的百度API_KEY"
SECRET_KEY = "你的百度SECRET_KEY"
BAILIAN_API_KEY = "你的百炼API_KEY"

3. 安装后端依赖

进入 server 目录：

cd server
pip install -r requirements.txt

4. 启动 Flask 后端
python app.py

正常情况下终端会显示类似：

* Running on http://127.0.0.1:5000
* Running on http://192.168.x.x:5000

请记下你的电脑当前局域网 IP。

5. 导入微信小程序项目

打开微信开发者工具，导入项目中的：

miniprogram
6. 修改前端请求地址

请将前端代码中的后端地址改成你自己电脑当前的局域网 IP，例如：

http://192.168.1.100:5000

通常需要检查这些接口地址：

/parse_bill
/speech_to_text
/analyze_bills

注意：
真机测试时不能使用 127.0.0.1，因为手机上的 127.0.0.1 指向的是手机本机，而不是你的电脑。

7. 真机测试

由于项目包含录音功能，建议使用真机测试。

测试时请确保：

手机和电脑连接同一个 Wi-Fi
Flask 后端保持运行
小程序中已填入正确的局域网 IP
电脑防火墙未拦截 5000 端口

## AI 能力说明
1. AI 分类

系统会将用户输入的消费描述发送给大模型进行分类判断。

输入示例：

酸菜鱼30元
雪糕3元
茶瀑布18元

输出示例：

餐饮
购物
交通
其他
2. AI 分析

系统不会直接把所有原始账单文本交给模型，而是先由程序完成结构化统计，再交由 AI 进行自然语言分析与建议生成。

这样做的优点：

结果更稳定
不容易编造数据
更适合工程实现
## 常见问题
1. 为什么真机请求失败？

常见原因：

前端仍写的是 127.0.0.1
局域网 IP 填错
手机和电脑不在同一 Wi-Fi
Flask 服务未启动
防火墙拦截了 5000 端口

2. 为什么录音成功但没有识别出文字？

请优先使用真机测试。
微信开发者工具模拟器中的录音可能无声或不稳定。

3. 为什么 AI 分类失败？

可能原因包括：

百炼 API Key 无效
账号没有模型调用权限
免费额度不足
模型名配置错误
接口调用异常

系统在这种情况下会自动回退到基础规则分类。

4. 为什么 AI 消费分析失败？

可能原因包括：

百炼接口调用失败
返回格式不符合 JSON 预期
网络异常

当前后端中已加入失败回退逻辑，至少会返回基础分析结果。

5. 为什么账单统计为空？

本项目当前使用本地存储保存账单。
首次运行时没有历史数据，需要先记录几笔账单后才能看到统计结果。

请不要把真实的 server/config.py 提交到 GitHub。

仓库中只应保留：

server/config.example.py

而不应提交：

server/config.py

如果你曾经误提交真实密钥，请立即：

去对应平台重置密钥
删除仓库中的明文内容
检查 Git 历史提交中是否仍然存在泄露记录
