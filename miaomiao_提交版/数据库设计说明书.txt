1. 引言
1.1 编写目的

本文档用于说明“喵喵记账”小程序的数据结构设计、数据组织方式以及数据字段定义，为后续数据库扩展、后端持久化与云端同步提供设计依据。

2. 当前数据存储方式

当前版本采用：

微信小程序本地存储

即：

wx.setStorageSync()
wx.getStorageSync()

进行账单保存。

3. 数据实体设计
3.1 Bill（账单实体）
实体说明

用于保存用户单笔消费记录。

字段设计
字段名	类型	说明
id	String	唯一标识
type	String	收入/支出
amount	Float	金额
category	String	消费分类
remark	String	消费备注
createdAt	String	创建时间
数据示例
{
  "id":"17123456789",
  "type":"支出",
  "amount":18,
  "category":"餐饮",
  "remark":"奶茶",
  "createdAt":"2026-04-20 14:32"
}
3.2 AIAnalysis（AI 分析实体）
实体说明

保存 AI 生成的消费分析内容。

字段设计
字段名	类型	说明
analysis	String	消费分析
suggestion	String	消费建议
4. 数据关系设计

当前版本主要为：

一个用户
    │
    └── 多条账单

即：

User 1 —— N Bill

由于当前版本未引入用户系统，因此默认：

本地设备对应一个用户数据空间
